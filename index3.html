Checkout Assumptions

Last package is implemented in production (not waiting approval)

Last stream and package version in production matches its corresponding in repository reademe file and directory

Steps if you don't have already cloned repository


Sign in gitlab
Open project page
Copy clone URL,Adjust it as in the example
Example :


You have to modify the link
from
git@SC-172-20-150-136.RegionOne.stc.corp:developers2/stc_bms_process_stream.git
to
git@172.20.150.136:developers2/stc_bms_process_stream.git
Now you need to make a local branches to track the production remote branch and run git pull to check the latest update
git branch

git checkout -b production origin/production
git pull

Now you have the latest package in your local directory

Steps if you've already cloned repository


git checkout -b production origin/production
if you did this command before just change it to
git checkout production
git pull

Now you have the latest package in your local directory
