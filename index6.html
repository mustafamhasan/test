Updating Gitlab Through releases

After finalizing the new features that will go to production ,and adjusting the packages convention and names
A release branch will be used to gather all this information to make the release similar to production then merge it and update

git checkout develop

git checkout -b rel_no

If you don't have all the release changes just publish the release branch so other developers could update it

Also for others developers to review before update


git push -u origin rel_no
git checkout master
Then Merge the updates to the master branch
git checkout master
git merge rel_no

Add tag to the release and push it to master branch
git tag -a v8 -m 'Production version 8'
git push -u origin master
git push --tags

Back Merging to develop branch
git checkout develop
git merge rel_no
git push -u origin develop
